# Pet Halo

Pet Halo is an independent, unofficial macOS companion that aims to display a transparent, always-on-top usage halo near Codex Pet. It does not modify Codex Desktop or Codex Pet.

> Pet Halo is an independent, unofficial project for Codex. It is not affiliated with or endorsed by OpenAI.

## Status

**M8 Release UI Polish is implemented and in final validation on top of the completed M7 Pet Ring contract.** Pet Halo still requests Accessibility only after an explicit enable action. Once enabled, a uniquely resolved Pet attaches immediately without a saved Pet anchor or first-use calibration. The next planned milestone is M9 — Public Beta Release Readiness; it has not started.

The target hierarchy remains Pet first, calibrated Codex standard window second, and free-floating Halo last. When Pet is unavailable, Halo stays hidden by default; following the Codex window must be enabled explicitly from the menu. Pet core selection prefers the unique stable near-square `AXSystemDialog` and retains the near-square `AXDialog` form only as a compatibility fallback. Discovery and tracking remain based on the raw AX frame center; the complete Ring panel uses that center plus one persisted fixed visual-center offset. `Adjust Ring Center` temporarily enables dragging and nudging with a center crosshair, then Save, Cancel, or Reset returns the panel to normal click-through behavior. Visual containment is a manual acceptance check, not an inferred AX guarantee. The legacy normalized Pet anchor remains deleted and the M4 window anchor remains separate.

Pet targeting selects only a transparent `448×252` vector surface whose midpoint remains locked to the Pet visual center, with independent Weekly, optional exact-300-minute 5h, and Today-versus-peak rings retaining their original radii around a `162`-point transparent center. The extra transparent width holds horizontal compact capsules outside the rings. Capsules retain the fixed Weekly/5h/Today identity-dot palette, use appearance-aware high-contrast key text and system value text, and expose stale/unavailable states without color alone. Their 24-point stack recenters when an optional metric is omitted. Dialog geometry still chooses only the partial-arc opening; the selected display's `visibleFrame` separately chooses the safe capsule side, including negative-coordinate displays, without moving the Ring center or persisting a display identity.

Pet movement callbacks wake one latest-value, display-linked follower in normal mode. While active, it samples only the already-selected Pet core geometry on each display refresh and stops after four unchanged refreshes; it never enumerates or reselects targets from that path. Medium continuous moves apply the newest target exactly without replaying intermediate frames; moves within `1.25 pt` and discontinuities of at least `96 pt` snap immediately. Reduce Motion applies every new Pet position directly, and a runtime setting change invalidates an active display-link callback. A missing event-path geometry sample is retried after `16 ms`; a bounded recovery health check also detects Codex process replacement or a silently changed selected frame without waiting for a message event. Dialog orientation alone retains its separate `180 ms` debounce. No queued window or decorative animation is added.

The M3 non-activating panel behavior remains intact: compact is click-through outside M4 window calibration, expanded accepts scrolling outside Pet targeting, and neither mode can become key or main. M8 adds an original abstract macOS AppIcon set and a separate monochrome template menu icon; neither uses Codex Pet or OpenAI/ChatGPT artwork. The M8-focused gate passes 105 Swift tests and two deterministic icon-asset checks; full regression, universal Release, retained M2–M7 smoke, and direct visual/live-Pet acceptance also pass. Draft PR CI is the final closeout gate.

M0 protocol feasibility remains complete with **PASS-CORE / PARTIAL-OPTIONALS**. The validated future MVP uses capability-driven data:

- weekly rate-limit remaining percentage and reset time as the primary halo;
- the Halo-owned app-server connection state;
- account usage summary and daily usage in an expanded view;
- a five-hour rate-limit segment only when a 300-minute window is returned;
- a Context segment only when a supported shared thread source and target selection are available.

Missing optional capabilities are unavailable and are never estimated. The bridge is fail-closed to the exact validated Codex CLI version `0.145.0-alpha.18`; unsupported versions do not launch an app-server. Protocol results must come from schemas generated by the current local Codex CLI, while production code uses only small hand-owned DTOs and never bundles those schemas. The project does not read or depend on Codex internal SQLite databases.

## Development commands

```sh
make bootstrap
make generate
make build
make test
make m0-tests
make m2-tests
make m2-smoke
make m3-tests
make m3-smoke
make m4-tests
make m4-smoke
make m5-tests
make m5-smoke
make m6-tests
make m6-smoke
make m7-tests
make m7-smoke
make m8-tests
make m8-smoke
make check
```

XcodeGen 2.46.0 is required. `project.yml` is the editable project source of truth; the generated Xcode project is committed and must not be hand-edited.

`make m2-tests` through `make m8-tests` are deterministic and require neither Accessibility permission nor a running Codex application. `make m8-smoke` checks the M8 visual/accessibility policies and compiled icon bundle; retained live interaction remains covered by `make m7-smoke`.

M8 is limited to release UI/accessibility polish, safe capsule placement, Reduce Motion behavior, original icons, and spacing; it adds no themes or decorative effects. M9 owns compatibility hardening, packaging, signing/notarization, privacy audits, release automation, and public-beta readiness. Completing M8 does not authorize M9.

See [the M8 report](docs/milestones/m8-release-ui-polish.md), [visual-accessibility ADR](docs/adr/0010-release-visual-accessibility.md), [compatibility contract](docs/COMPATIBILITY.md), and [privacy policy](docs/PRIVACY.md).
